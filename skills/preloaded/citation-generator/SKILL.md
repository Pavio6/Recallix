# 引用生成器

在基于知识库、文档或外部资料作答时，生成清晰、准确、可追溯的引用信息。

## 适用场景

当用户有以下需求时使用本技能：

- 生成参考文献
- 为回答中的事实标注来源
- 给知识库检索结果添加出处
- 将已有资料整理为规范引用格式
- 明确要求“给出引用”“注明来源”“附参考文献”

## 工作原则

1. **先核对来源，再生成引用**  
   只引用已经提供或已经检索到的真实来源，不凭空补全作者、年份、页码或链接。

2. **引用紧跟事实**  
   哪一句使用了来源，引用就尽量放在哪一句之后，避免把所有出处堆到结尾导致对应关系不清。

3. **优先保留可追溯信息**  
   能获得文档名、作者、年份、章节、页码、段落、URL、检索块 ID 时，应尽量保留。

4. **同一回答内格式统一**  
   默认使用一种格式贯穿全文；如果用户指定 APA、MLA、Chicago、GB/T 7714 或其他格式，则优先服从用户要求。

5. **不伪造引用**  
   缺失的信息要明确标注“未提供”或改用现有信息表达，不要猜测。

## 默认输出方式

### 1. 文中引用

对知识库或内部文档，默认使用简洁格式：

```text
[文档名称，第 X 页]
[文档名称，段落 X]
[文档名称，章节 X]
```

示例：

```text
年假申请需要至少提前 3 个工作日提交。[员工手册2024.pdf，第 15 页]
```

如果同一句同时依据多个来源，可并列：

```text
[员工手册2024.pdf，第 15 页；请假制度说明.docx，第 2 节]
```

### 2. 参考文献列表

当用户要求参考文献，或回答中引用较多时，在末尾追加：

```markdown
## 参考文献

1. 文档名称. 章节/页码/段落信息.
2. 作者. 年份. 标题. 来源.
```

## 常见格式

### APA

```text
作者. (年份). 标题. 来源.
```

### MLA

```text
作者. “标题.” 来源, 年份.
```

### Chicago

```text
作者. 标题. 出版地: 出版者, 年份.
```

### GB/T 7714

```text
[序号] 作者. 题名[文献类型]. 出版地: 出版者, 年份.
```

当用户明确要求 **GB/T 7714**，或引用信息涉及中文学术文献、图书、期刊、学位论文时：

1. 先读取 `references/gbt7714.md`
2. 再按其中的文献类型规则生成最终条目

如果用户提供了多条结构化参考文献草稿，且需要统一编号、字段顺序或批量规范化，可执行：

```text
scripts/normalize_references.py
```

该脚本通过标准输入接收 JSON 数组，输出统一整理后的参考文献文本。

## 处理流程

1. 提取可用的来源信息：标题、作者、年份、页码、章节、段落、链接等。
2. 判断用户是否指定引用格式；未指定时，内部知识库默认用简洁格式。
3. 在每个被来源支持的事实后添加文中引用。
4. 如有需要，在结尾汇总参考文献列表。
5. 检查引用是否与正文事实一一对应，删除无关或重复引用。

## 特殊情况

### 来源信息不完整

- 无页码：使用章节、段落、分块编号或标题定位。
- 无作者：以机构名或文档名代替。
- 无年份：可写“无日期”或仅保留已有信息。

### 同一来源被多次使用

- 文中可多次标注。
- 参考文献列表中只保留一条完整记录。

### 用户只要“引用信息”，不要正文

直接输出规范化后的引用条目，不额外扩写内容。

## 禁止事项

- 不得编造不存在的来源
- 不得让引用与正文事实错配
- 不得在未核实的情况下补写页码、作者或出版信息
- 不得混用多种格式而不加说明
