# Recallix 前端开发文档

## 1. 前端目标

前端使用 `React` 实现，为 `Recallix` 提供一个简洁、可上线的 Web 界面。

当前版本只实现两条核心业务：

1. 知识库
   - 用于上传文档

2. 对话
   - 用于基于 RAG 的聊天问答

当前版本还需要支持：

1. 用户注册
2. 用户登录
3. 登录态保持
4. 左侧展示最近 7 天会话
5. 只支持文本输入聊天

当前版本暂不实现：

1. MCP
2. Skills
3. Agent 模式
4. 语音输入
5. 图片输入
6. 文件拖入聊天框
7. 多模态问答
8. Wiki、共享空间、搜索页等额外模块

---

## 2. 技术选型

1. 前端框架
   - `React`

2. 构建工具
   - `Vite`

3. 路由
   - `React Router`

4. 状态管理
   - 第一版优先使用：
     - `React Query`
     - `Context`
   - 如果后续状态明显变复杂，再考虑 `Zustand`

5. 请求库
   - `Axios`

6. 样式
   - 可使用 `Tailwind CSS` 或普通 CSS Modules
   - 第一版重点是布局清晰和交互完整，不追求复杂视觉效果

7. 图标
   - 推荐使用 `lucide-react`

---

## 3. 页面结构

前端共 4 类页面：

1. 登录页
2. 注册页
3. 主应用页
4. 404 页面

路由建议：

```text
/login
/register
/app/chat
/app/knowledge
```

登录后默认进入：

```text
/app/chat
```

---

## 4. 登录页

### 4.1 页面目标

让已有用户登录系统，并拿到访问令牌。

### 4.2 页面元素

1. 项目名
   - `Recallix`

2. 表单
   - 邮箱输入框
   - 密码输入框

3. 操作按钮
   - 登录

4. 辅助入口
   - 跳转注册页

### 4.3 交互要求

1. 表单校验
   - 邮箱不能为空
   - 邮箱格式合法
   - 密码不能为空

2. 登录成功
   - 保存 `access_token`
   - 保存 `refresh_token`
   - 跳转 `/app/chat`

3. 登录失败
   - 展示接口返回的错误信息

---

## 5. 注册页

### 5.1 页面目标

让新用户创建账号。

### 5.2 页面元素

1. 项目名
   - `Recallix`

2. 表单
   - 邮箱
   - 昵称
   - 密码
   - 确认密码

3. 操作按钮
   - 注册

4. 辅助入口
   - 跳转登录页

### 5.3 交互要求

1. 校验
   - 邮箱格式合法
   - 密码不能为空
   - 两次密码一致

2. 注册成功
   - 跳转登录页

3. 注册失败
   - 展示接口错误

## 7. 左侧边栏

### 7.1 边栏结构

左侧边栏分成 4 块：

1. 顶部品牌区
   - `Recallix`

2. 主导航
   - `知识库`
   - `对话`

3. 最近 7 天会话列表
   - 仅在“对话”模块中重点使用

4. 底部用户区
   - 当前用户昵称或邮箱
   - 退出登录

### 7.2 导航规则

1. `知识库`
   - 点击进入 `/app/knowledge`

2. `对话`
   - 点击进入 `/app/chat`

3. 当前选中项高亮

### 7.3 最近 7 天会话

只展示最近 7 天有消息的 session。

每条 session 显示：

1. 会话标题
2. 最近更新时间

交互：

1. 点击某个 session
   - 打开对应历史聊天

2. 新建会话
   - 在“对话”导航旁提供 `+` 按钮
   - 点击后创建空白 session，并进入新聊天

3. 会话为空时
   - 显示简洁空状态

第一版暂不需要：

1. 会话分组
2. 搜索会话
3. 删除会话
4. 重命名会话

---

## 8. 知识库页面

### 8.1 页面目标

让用户上传用于 RAG 的文档。

### 8.2 页面内容

1. 页面标题
   - `知识库`

2. 上传区域
   - 支持点击上传
   - 支持选择文件

3. 文档列表
   - 文件名
   - 文件大小
   - 上传时间
   - 解析状态

### 8.3 支持格式

第一版只支持：

1. `.md`
2. `.txt`
3. `.docx`

### 8.4 上传交互

1. 用户选择文件
2. 前端校验格式
3. 调用上传接口
4. 上传成功后刷新文档列表
5. 展示文档解析状态：
   - 处理中
   - 成功
   - 失败

### 8.5 第一版边界

第一版暂不做：

1. 文件夹上传
2. 批量删除
3. 在线预览
4. 标签管理
5. 多知识库切换

如果后端第一版已经支持多个知识库，前端也先只展示一个默认知识库即可。

## 9. 对话页面

### 9.1 页面目标

让用户基于知识库进行文本问答。

### 9.2 页面布局

右侧主内容区分为：

1. 顶部标题区
   - 标题：`智能问答`

2. 中间消息区
   - 展示用户消息和助手消息

3. 底部输入区
   - 文本输入框
   - 发送按钮

当当前 session 还没有消息时，页面中部可展示：

1. 主标题
   - `基于知识库内容问答`

2. 简短提示
   - 告诉用户可以直接提问

不需要实现示例问题气泡，也不需要模型选择器。

### 9.3 当前聊天能力

当前只有一条聊天链路：

```text
RAG 问答
```

也就是说：

1. 用户输入文本问题
2. 后端走 RAG 提问链路
3. 前端接收流式回答
4. 展示最终答案

当前不做：

1. 普通聊天模式切换
2. Agent 模式切换
3. 模型切换
4. 联网搜索开关
5. 文件选择器
6. 图片上传
7. 语音输入

### 9.4 消息展示

用户消息：

1. 右侧显示
2. 仅文本

助手消息：

1. 左侧显示
2. 支持 Markdown 渲染
3. 支持流式逐步展示

### 9.5 发送消息流程

1. 用户输入问题
2. 点击发送或按 Enter
3. 如果当前没有 session，则先创建 session
4. 立即在前端插入用户消息
5. 调用聊天接口
6. 接收 SSE 流式结果
7. 实时追加助手消息内容
8. 回答完成后刷新左侧最近会话列表

### 9.6 输入框要求

1. 只支持文本输入
2. 支持多行
3. `Enter` 发送
4. `Shift + Enter` 换行
5. 发送中禁用重复提交
6. 空内容不可发送

## 10. 接口依赖

### 10.1 认证

1. `POST /auth/register`
2. `POST /auth/login`
3. `POST /auth/refresh`
4. `POST /auth/logout`

### 10.2 知识库

1. `GET /knowledge-bases`
2. `POST /knowledge-bases/:id/files`
3. `GET /knowledges`

### 10.3 会话

1. `POST /sessions`
2. `GET /sessions/recent?days=7`
3. `GET /sessions/:id/messages`

### 10.4 对话

1. `POST /sessions/:id/chat`
   - 需要支持流式返回

## 12. 登录态与请求处理

1. 登录成功后保存 token
2. 请求拦截器自动带上 `Authorization`
3. `access_token` 过期后，尝试用 `refresh_token` 刷新
4. 刷新失败则跳回登录页
5. 未登录用户访问 `/app/*`
   - 自动重定向 `/login`

---

## 13. 页面状态要求

所有核心页面都要有以下状态：

1. 加载中
2. 空状态
3. 错误状态
4. 成功状态

重点：

1. 上传文件后，要明确显示解析中
2. 聊天流式生成时，要明确显示正在回答
3. 会话列表没有数据时，不要留白

---

## 14. 第一版开发顺序

1. 搭建 React 项目骨架
2. 完成登录页和注册页
3. 完成主布局和左侧边栏
4. 完成知识库页面上传和文件列表
5. 完成对话页基础布局
6. 接入 session 创建与最近 7 天 session 列表
7. 接入聊天流式回答
8. 补齐 loading、empty、error 状态

---

## 15. 最终页面闭环

```text
注册 / 登录
-> 进入主应用
-> 上传知识库文档
-> 等待文档解析完成
-> 新建对话
-> 输入文本问题
-> RAG 流式回答
-> 左侧保留最近 7 天 session
```

这就是 `Recallix` 第一版前端需要完整打通的主流程。
